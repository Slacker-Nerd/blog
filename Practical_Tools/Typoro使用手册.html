# Typora使用手册

Typora   Typora 是一款跨平台的 Markdown 编辑器软件，支持 Markdown 的标准语法，同时这款软件还支持动态预览功能，一键预览，是一款不可多得的优质 Markdown 编辑器。

MarkDown   Markdown 是一种轻量级标记语言，创始人为约翰·格鲁伯(John Gruber)。 它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的 XHTML(或者HTML) 文档。这种语言吸收了很多在电子邮件中已有的纯文本标记的特性，由于 Markdown 的轻量化、易读易写特性，并且对于图片，图表、数学式都有支持，许多网站都广泛使用 Markdown 来撰写帮助文档或是用于论坛上发表消息。

# 二、文档

# 文本格式设置

1、标题：用 #数量对应 1 - 6 级标题，也可使用快捷键 Ctrl+1 至 Ctrl+6 快速设置。示例：

# 一级标题
## 二级标题
### 三级标题

2、文字样式

样式 语法 快捷键
加粗 加粗内容或__加粗内容__ Ctrl+B/Command+B
倾斜 倾斜内容或_倾斜内容_ Ctrl+I/Command+I
删除线 删除线内容 Shift+Alt+5
下划线 下划线内容 Ctrl+U/Command+U

3、分割线：输入---或***并回车，即可生成分割线，用于分隔文档段落。

常用内容编辑

# （一）列表制作

  1. 无序列表：输入-、*、+符号加空格，回车继续添加下一项，按 Backspace 退出编辑。示例：

    - 列表项1
- 列表项2
  - 嵌套子项

  2. 有序列表：输入数字 + . + 空格（如 1. ），回车自动生成下一个编号，支持嵌套子列表。示例：

    1. 第一项
2. 第二项
   1. 子项1
   2. 子项2

  3. 任务列表：输入 - + 空格 + [空格] + 空格创建未完成任务，将括号内改为 x 即为已完成，示例：

    - [ ] 待办事项1
- [x] 已完成事项

# （二）链接与图片

  1. 超链接：常用行内式，格式为[链接文字](链接地址 "可选标题")。选中文本按 Ctrl+K 可弹出对话框快速添加，示例：[Typora官网](https://typoraio.cn/ "官方网站")。
  2. 图片：格式为![替代文字](图片路径/URL "可选标题")。本地图片直接拖拽到编辑区会自动生成代码，网络图片粘贴 URL 即可。

# （三）引用与代码块

  1. 引用：输入 > +空格后写内容，支持嵌套，多一个 > 就是下一级引用，示例：

    > 一级引用
>> 二级引用

  2. 代码展示

    • 行内代码：用反引号包裹，比如print("Hello")。
    • 代码块：输入后加编程语言名称（如python），回车编写代码，再输结束，可实现语法高亮，示例：
    def hello():
    print("Typora")

# （四）表格

  1. 快速创建：按下 Ctrl+T 弹出对话框选择行列数，或手动输入语法，回车后生成表格，示例：

    | 姓名 | 年龄 |
| ---- | ---- |
| 张三 | 20 |

  2. 便捷调整：鼠标拖动表格边框可调整列宽，选中单元格右键可进行增删行列操作。

# 辅助工具

# 转换工具

项目原地址：microsoft/markitdown：用于将文件和办公文档转换为Markdown的Python工具。 (opens new window)

安装

安装 MarkItDown 时，请使用 pip： 。或者，您也可以从源代码安装：pip install 'markitdown[all]'

git clone git@github.com:microsoft/markitdown.git
cd markitdown
pip install -e 'packages/markitdown[all]'

用法

  • 命令行（将path-to-file换成你的文件名）

    markitdown path-to-file.pdf > document.md

  • python代码

    '''
安装指令：pip install markitdown[all]
'''
from markitdown import MarkItDown

txt="python二级（24年9月）"

# 创建 MarkItDown 实例
md = MarkItDown(enable_plugins=False)
# 将 PDF 文件转换为 Markdown
result = md.convert(txt+".pdf")
# 获取转换后的文本内容
md_content = result.text_content
# 将转换后的内容保存为 同名.md 文件
with open(txt+ ".md", "w") as md_file:
    md_file.write(md_content)
print("PDF 转换为 Markdown 文件成功！")

# 安装

首先，我们需要安装 PicGo 软件与 Gitee-uploader 插件。

打开 PicGo 官网下载地址 (opens new window) ，下载安装包。下载完成直接安装即可。

# 配置

1.新建仓库

2.输入仓库名称，设置模板选择 Readme 文件，选择分支模型选择单分支模型（只创建 master 分支），然后点击创建

3.创建完毕后，点击管理选项确定配置相同

4.然后依次点击个人头像 => 设置 => 私人令牌 => 生成新令牌（生成的私人令牌要保存好，之后的操作要用到）

5.在图床设置中选择 gitee，把私人令牌填入token，repo为打开仓库后网页链接com/后面的（如果找不到可以在插件设置里下载gitee-uploader 1.1.2）

（1）repo：用户名/仓库名称，找不到的可以直接复制仓库的url；

（2）branch：分支，这里写上master

（3）token：填入码云的私人令牌

（4）path：路径，一般写上images

（5）customPath：提交消息，这一项和下一项customURL都不用填。 在提交到码云后，会显示提交消息，插件默认提交的是 Upload 图片名 by picGo - 时间

# 管理图库

第一步：克隆 Gitee 仓库到本地

  1. 安装 Git（若未安装：https://git-scm.com/downloads，一路默认安装即可）；

  2. 登录 Gitee，进入目标仓库，复制仓库的「HTTPS 地址」（仓库主页 → 克隆 / 下载 → 复制 HTTPS 链接，比如 https://gitee.com/你的用户名/仓库名.git）；

  3. 本地新建一个空文件夹（比如 D:\gitee-repo），在该文件夹下打开命令窗口；

  4. 执行克隆命令（粘贴刚才复制的链接）：

    git clone https://gitee.com/你的用户名/仓库名.git
    • 按提示输入 Gitee 账号密码（若提示「认证失败」，可改用「私人令牌」：Gitee 设置 → 私人令牌 → 生成令牌，克隆时用户名填账号，密码填令牌）；
    • 克隆完成后，本地文件夹会出现仓库的所有文件。

第二步：本地处理文件（删除 / 修改 / 整理）

打开克隆后的本地仓库文件夹，按需操作：

  • 删除文件：直接右键删除无用的文件 / 文件夹（比如旧图片、错误上传的文件）；
  • 修改文件：编辑需要调整的文件（比如重命名图片、替换新版本图片）；
  • 新增文件：把需要上传的新文件复制到对应目录；
  • ✅ 建议：处理后检查一遍，确保本地文件是最终需要的状态（避免误删）。

第三步：将处理后的文件推送到 Gitee（覆盖远程）

回到「Git Bash Here」窗口，执行以下命令（按顺序）：

  1. 查看本地修改状态（可选，确认处理的文件）：

    git status

    会显示「删除的文件、新增的文件、修改的文件」，确认无误后继续。

  2. 将所有修改加入暂存区：

    git add .  # 注意：add 后面有一个英文句号，表示提交所有修改（删除/新增/修改）
    • 若只想提交部分文件，可指定路径：git add 文件名/文件夹名。

  3. 提交修改（填写备注，方便追溯）：

    git commit -m "清理仓库：删除无用图片，更新最新文件"  # 备注可自定义，清晰即可

  4. 推送到 Gitee 远程仓库：

    git push origin master  # 若仓库默认分支是 main，替换为 git push origin main

    • 若推送失败（提示「本地分支落后于远程」），先拉取最新代码再推送：

      git pull origin main  # 拉取远程最新内容
git push origin main  # 再次推送

第四步：验证结果

登录 Gitee 仓库主页，刷新页面，会看到远程仓库的文件已和本地处理后的状态完全一致（删除的文件消失，新增 / 修改的文件同步上线）。
